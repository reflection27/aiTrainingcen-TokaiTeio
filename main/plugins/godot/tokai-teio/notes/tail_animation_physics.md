# 尾巴动画物理：为什么用 cosine 而不是摆线？

**Q：下落段为什么用 cosine 模拟摆线，而不是直接用摆线？**

---

## 1. 摆线不适合作为尾巴下落的物理模型

摆线（cycloid）描述的是「物体沿摆线形轨道滑落」的最速路径（brachistochrone）。尾巴下落不是沿轨道滑动，而是像摆锤一样绕关节旋转。适合摆锤的物理模型是简谐运动（SHM），其方程正好是 cosine。

## 2. 摆线等时降的运动方程本身就是 cosine

这是摆线最反直觉的性质：摆线轨道之所以是等时降（tautochrone），恰恰是因为沿弧长的恢复力始终正比于位移，使运动满足 SHM 方程：

```
θ''(t) + ω²θ(t) = 0  →  θ(t) = θ₀·cos(ωt)
```

所以「用 cosine 近似摆线」这个说法本身就有误——**cos 就是摆线等时降的精确解，不是近似**。

## 3. 直接用摆线参数曲线作动画曲线的技术问题

如果想把摆线的几何形状 `x = r(θ-sinθ), y = r(1-cosθ)` 硬套成「时间→角度」曲线，需要以归一化时间 `s` 反解 `θ`：

```
s = (θ - sinθ) / 2π
```

这是超越方程，没有解析解，只能 Newton 迭代——每帧跑迭代不合适。

---

## 结论

代码中下落段注释应为：

```gdscript
# 下落：cos 曲线 —— 摆锤 SHM 精确解，顶慢底快，模拟重力感
return amp * cos(lf / T_FALL * PI * 0.5)
```
